# Build a Core Banking Prototype with IBM Bob

Welcome!

Use IBM Bob to build a prototype core banking system end to end: **understand** a Python/FastAPI backend and its multiple user roles through documentation and architecture diagrams, **build** a modern React UI fast with a Carbon-style MCP, then **find and patch** the security flaws.


## Setup

**Node.js 18 or later** is required for this lab.

Check whether you already have it:

```bash
node --version
```

If the version is 18 or above you're set. Otherwise, download and run the LTS installer for your platform from **[nodejs.org/en/download](https://nodejs.org/en/download)**, then verify:

```bash
node --version   # v18.x.x or higher
```

## What to expect

A hands-on, guided lab in four parts.

| Part | What you do |
|---|---|
| **[1 · Explore the Codebase](<01_explore_the_codebase/part_1_explore_the_codebase.md>)** | Chat with Bob to map the FastAPI core-banking backend and its user roles, generating architecture docs and diagrams. |
| **[2 · Build the Teller App](<02_build_the_teller_app/part_2_build_the_teller_app.md>)** | Spec-drive a React + IBM Carbon teller UI using the UI style MCP server and wired to the live backend. |
| **[3 · Security Audit & Fixes](<03_fix_security_issues/part_3_security_audit.md>)** | Have Bob discover and remediate the security flaws in the banking code. |
| **[4 · Extend](<02_build_the_teller_app/part_4_extend_and_harden.md>)** *(optional)* | Extend the Part 2 app — tests, a new feature, accessibility, resilience, or containerization. |

> Start with **[Part 1 →](<01_explore_the_codebase/part_1_explore_the_codebase.md>)**.
